Database Design: Renting A Car Project

My group and I created a database for a hypothetical car rental company to track operations, such as rental car inventories, rental contracts, and billing. We created the Entity-Relationship (ER) Diagram to demonstrate the relationships between entities and classify the types of relationships between those entities. We created the Diagram Schema to determine if there are any entities that are dependent on another entity, and to figure out the primary and foreign keys for the actual database creation in Oracle SQL Developer. After figuring out these details, we began writing SQL code to create our RentACar database. We created the tables, primary key and foreign keys for each table, demonstrated each relationship between entities and tables, and then populated each table with data. We updated and deleted data from the table as needed. Our database was ready for testing and querying.
